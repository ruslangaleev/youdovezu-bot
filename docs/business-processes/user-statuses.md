# Статусы пользователей

## Общее описание

Данный документ описывает различные статусы пользователей в системе и их возможности в зависимости от статуса.

**Примечание:** Для понимания ролей в системе см. раздел [Роли в системе](./system-roles.md).

## Статусы пользователя

### Пользователь в процессе регистрации

**Характеристики:**
- Не завершил все шаги регистрации
- Видит только экран завершения регистрации
- Не может использовать функции приложения

**Ограничения:**
- Нет доступа к главному меню
- Не может создавать запросы на поездки
- Не может искать поездки
- Не может загружать документы

### Активный пользователь

**Характеристики:**
- Завершил регистрацию
- Согласился с политикой конфиденциальности
- Имеет доступ к функциям приложения

**Возможности:**
- Доступ к главному меню
- Публикация запросов на поездки (Requester)
- Просмотр списка своих запросов
- Редактирование своих запросов
- Удаление своих запросов
- Загрузка документов для получения доступа к просмотру запросов (Offerer)

**Отображение статуса:**
- В главном меню отображается статус пользователя
- Если документы одобрены и активен триал — отображается "(Триал)"

### Пользователь с одобренными документами (Offerer)

**Характеристики:**
- Завершил регистрацию
- Загрузил документы (ВУ, СТС, аватарка)
- Документы проверены и одобрены администратором
- Может быть как Пользователь, так и Администратор

**Возможности:**
- Все возможности активного пользователя
- Просмотр опубликованных запросов от других пользователей
- Предложение своих услуг с указанием цены
- Получение одобрений от пользователей, которые ищут место

**Триал:**
- 14 дней бесплатного использования после одобрения документов
- Отображается как "(Триал)" в статусе пользователя

### Заблокированный пользователь

**Характеристики:**
- Заблокирован администратором
- Доступ к приложению запрещен

**Ограничения:**
- Нет доступа к приложению
- Все функции недоступны

## Переходы между статусами

### От нового пользователя к активному пользователю

1. Пользователь соглашается с политикой конфиденциальности
2. Получает доступ к главному меню
3. Получает возможность публиковать запросы на поездки (Requester)

### От активного пользователя к пользователю с одобренными документами

1. Пользователь загружает документы (ВУ, СТС, аватарка)
2. Документы отправляются на проверку
3. Администратор проверяет документы
4. Если документы одобрены:
   - Пользователь получает доступ к просмотру запросов других пользователей (Offerer)
   - Начинается триальный период (14 дней)
   - Может просматривать запросы и предлагать свои услуги

## Отображение статусов

### В главном меню

Пользователь видит свой статус внизу главного меню:

- **Статус роли:** Отображается функциональная роль (Requester / Offerer)
- **Триал:** Отображается "(Триал)" если активен триальный период

### Примеры статусов

**Активный Requester:**
- Завершил регистрацию
- Согласился с политикой конфиденциальности
- Может публиковать запросы на поездки

**Активный Offerer (триал):**
- Завершил регистрацию
- Документы проверены и одобрены
- Триал активен (14 дней)
- Может просматривать запросы и предлагать услуги

**Активный пользователь с обеими ролями:**
- Завершил регистрацию
- Может публиковать запросы на поездки (Requester)
- Может просматривать запросы других пользователей и предлагать услуги (Offerer)
