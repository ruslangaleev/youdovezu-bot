# Роли в системе

## Общее описание

YouDovezu — это P2P-сервис для организации совместных поездок в Республике Башкортостан. Платформа объединяет людей, которые едут в одном направлении, позволяя им делиться поездкой и расходами. Это не агрегатор перевозок, а сервис для поиска попутчиков.

В системе существуют два уровня ролей:

1. **Роли доступа** — определяют права пользователя в системе (Пользователь / Администратор)
2. **Функциональные роли** — определяют функции пользователя в поездках (Offerer / Requester)

## Роли доступа

### Пользователь

**Описание:**
Обычный участник платформы без прав модерации.

**Характеристики:**
- Имеет базовый доступ к функциям платформы
- Может публиковать запросы на поездки
- Может загружать документы для получения доступа к просмотру запросов других пользователей
- Не имеет доступа к функциям модерации

**Возможности:**
- Публикация запросов на поездки (Requester)
- Загрузка документов для получения доступа к просмотру запросов (Offerer)
- Просмотр своих запросов
- Управление своими запросами (создание, редактирование, удаление)

**Ограничения:**
- Не может модерировать документы других пользователей
- Не видит кнопку "Модерация" в главном меню

### Администратор

**Описание:**
Пользователь с расширенными правами, который может модерировать документы и предоставлять доступ к просмотру запросов.

**Характеристики:**
- Имеет все возможности обычного пользователя
- Дополнительно имеет доступ к функциям модерации документов
- Может проверять и одобрять/отклонять документы пользователей

**Возможности (дополнительно к возможностям пользователя):**
- Доступ к кнопке "Модерация" в главном меню
- Просмотр списка всех документов на проверке
- Просмотр детальной информации о документах
- Заполнение полей с информацией о водителе и автомобиле
- Одобрение или отклонение документов
- Предоставление доступа пользователям к просмотру запросов

**Особенности:**
- Администратор может быть как Offerer, так и Requester
- Администратор использует те же функции для создания запросов и просмотра предложений, что и обычный пользователь
- Администратор имеет дополнительный доступ к модерации документов

## Функциональные роли

### Offerer (Пользователь, который предоставляет автомобиль)

**Описание:**
Пользователь, который предоставляет свой автомобиль для совместной поездки и готов взять с собой попутчиков.

**Характеристики:**
- Имеет свой автомобиль
- Предоставляет транспортное средство для поездки
- Готов взять с собой попутчиков

**Возможности:**
- Просмотр опубликованных запросов от пользователей, которые ищут место в попутном авто
- Предложение своих услуг с указанием цены за место
- Получение одобрений от пользователей, которые ищут место
- Выполнение совместных поездок
- Также может создавать свои запросы на поездку (если сам ищет попутчиков)

**Условия получения:**
- Загрузка и проверка документов (водительское удостоверение, СТС, аватарка)
- После одобрения документов администратором получает доступ к просмотру запросов других пользователей

### Requester (Пользователь, который ищет место в попутном авто)

**Описание:**
Пользователь, который ищет возможность поехать вместе с другими попутчиками в их автомобиле.

**Характеристики:**
- Нужен транспорт для поездки
- Ищет попутчиков, которые предоставляют автомобиль
- Готов поделиться расходами на поездку

**Возможности:**
- Публикация запросов на поездку с указанием маршрута
- Управление своими запросами (создание, редактирование, удаление)
- Просмотр списка своих опубликованных запросов
- Получение предложений от пользователей, которые предоставляют автомобиль
- Одобрение или отклонение предложений
- Организация совместной поездки

**Условия получения:**
- Согласие с политикой конфиденциальности
- После согласия пользователь получает возможность публиковать запросы на поездки

## Сочетание ролей

И Пользователь, и Администратор могут иметь обе функциональные роли одновременно:

**Пользователь + Requester:**
- Обычный пользователь, который публикует запросы на поездки
- Ищет попутчиков, которые предоставляют автомобиль

**Пользователь + Offerer:**
- Обычный пользователь, который предоставляет автомобиль
- Просматривает запросы и предлагает свои услуги

**Администратор + Requester:**
- Администратор, который публикует запросы на поездки
- Имеет дополнительный доступ к модерации документов

**Администратор + Offerer:**
- Администратор, который предоставляет автомобиль
- Просматривает запросы и предлагает свои услуги
- Имеет дополнительный доступ к модерации документов

## Взаимодействие ролей

### Типичный сценарий

1. **Requester** создает и публикует запрос на поездку:
   - Указывает маршрут (откуда и куда)
   - Добавляет комментарий (если нужно)
   - Запрос становится видимым для Offerer

2. **Offerer** просматривает опубликованные запросы:
   - Видит запросы от Requester
   - Выбирает подходящий запрос
   - Предлагает свою услугу с указанием цены

3. **Requester** получает предложение:
   - Рассматривает предложение (цена, информация о попутчике)
   - Одобряет или отклоняет предложение

4. После одобрения:
   - Requester и Offerer связываются между собой
   - Договариваются о деталях поездки
   - Выполняют совместную поездку
   - Requester оплачивает свою часть расходов

## Важные особенности

### P2P-характер сервиса

- YouDovezu не является агрегатором перевозок
- Платформа только объединяет попутчиков
- Пользователи сами договариваются о деталях поездки
- Платформа не предоставляет услуги перевозки

### Совместные поездки

- Поездки организуются между равноправными участниками
- Это не услуга перевозки, а совместное использование автомобиля
- Расходы делятся между участниками
- Все участники едут в одном направлении

### Безопасность

- Все пользователи проходят регистрацию
- Offerer проходят проверку документов администратором
- Платформа обеспечивает безопасное взаимодействие между пользователями
