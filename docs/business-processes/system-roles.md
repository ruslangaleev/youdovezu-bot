# Роли в системе

## Общее описание

YouDovezu — это P2P-сервис для организации совместных поездок в Республике Башкортостан. Платформа объединяет людей, которые едут в одном направлении, позволяя им делиться поездкой и расходами. Это не агрегатор перевозок, а сервис для поиска попутчиков.

В системе существуют две основные роли пользователей:

## Offerer (Пользователь, который предоставляет автомобиль)

**Описание:**
Пользователь, который предоставляет свой автомобиль для совместной поездки и готов взять с собой попутчиков.

**Характеристики:**
- Имеет свой автомобиль
- Предоставляет транспортное средство для поездки
- Планирует поездку по определенному маршруту
- Готов взять с собой попутчиков

**Возможности:**
- Просмотр опубликованных запросов от пользователей, которые ищут место в попутном авто
- Предложение своих услуг с указанием цены за место
- Получение одобрений от пользователей, которые ищут место
- Выполнение совместных поездок
- Также может создавать свои запросы на поездку (если сам ищет попутчиков)

**Условия получения:**
- Загрузка и проверка документов (водительское удостоверение, СТС, аватарка)
- После одобрения документов получает доступ к просмотру запросов других пользователей

**Процесс работы:**
1. Пользователь в Telegram боте нажимает на кнопку "Поездки"
2. Открывается веб-приложение внутри Telegram
3. Пользователь нажимает кнопку "Предложить машину"
4. Загружает документы через веб-приложение (ВУ, СТС, аватарка)
5. Документы проверяются администратором
6. После одобрения пользователь получает доступ к просмотру запросов
7. Просматривает запросы пользователей, которые ищут место в попутном авто
8. Предлагает свои услуги с указанием цены
9. Получает одобрение от пользователей
10. Выполняет совместную поездку

## Requester (Пользователь, который ищет место в попутном авто)

**Описание:**
Пользователь, который ищет возможность поехать вместе с другими попутчиками в их автомобиле.

**Характеристики:**
- Нужен транспорт для поездки
- Ищет попутчиков, которые предоставляют автомобиль
- Готов поделиться расходами на поездку

**Возможности:**
- Публикация запросов на поездку с указанием маршрута
- Управление своими запросами (создание, редактирование, удаление)
- Просмотр списка своих опубликованных запросов
- Получение предложений от пользователей, которые предоставляют автомобиль (в будущем)
- Одобрение или отклонение предложений (в будущем)
- Организация совместной поездки (в будущем)

**Условия получения:**
- Согласие с политикой конфиденциальности
- После согласия пользователь получает возможность публиковать запросы на поездки

**Процесс работы:**
1. Пользователь в Telegram боте нажимает на кнопку "Поездки"
2. Открывается веб-приложение внутри Telegram
3. Пользователь нажимает кнопку "Ищу машину"
4. Создает запрос на поездку с указанием маршрута
5. Запрос публикуется и становится доступным для просмотра
6. Пользователи, которые предоставляют автомобиль, могут видеть запрос и предлагать свои услуги
7. Пользователь получает предложения с указанием цены
8. Пользователь рассматривает предложения и одобряет подходящее
9. После одобрения пользователь связывается с попутчиком и организует поездку

## Взаимодействие ролей

### Типичный сценарий

1. **Requester** создает и публикует запрос на поездку:
   - Указывает маршрут (откуда и куда)
   - Добавляет комментарий (если нужно)
   - Запрос становится видимым для Offerer

2. **Offerer** просматривает опубликованные запросы:
   - Видит запросы от Requester
   - Выбирает подходящий запрос
   - Предлагает свою услугу с указанием цены

3. **Requester** получает предложение:
   - Рассматривает предложение (цена, информация о попутчике)
   - Одобряет или отклоняет предложение

4. После одобрения:
   - Requester и Offerer связываются между собой
   - Договариваются о деталях поездки
   - Выполняют совместную поездку
   - Requester оплачивает свою часть расходов

## Важные особенности

### P2P-характер сервиса

- YouDovezu не является агрегатором перевозок
- Платформа только объединяет попутчиков
- Пользователи сами договариваются о деталях поездки
- Платформа не предоставляет услуги перевозки

### Совместные поездки

- Поездки организуются между равноправными участниками
- Это не услуга перевозки, а совместное использование автомобиля
- Расходы делятся между участниками
- Все участники едут в одном направлении

### Безопасность

- Все пользователи проходят регистрацию
- Offerer проходят проверку документов
- Платформа обеспечивает безопасное взаимодействие между пользователями

## Переход между ролями

Пользователь может иметь обе роли одновременно:

- **Только Requester:** Пользователь может публиковать запросы, но не может просматривать запросы других
- **Только Offerer:** Пользователь может просматривать запросы и предлагать услуги, но также может создавать свои запросы
- **Requester и Offerer:** Пользователь может и публиковать запросы, и просматривать запросы других, предлагая свои услуги

## Примечания

- Термины "Offerer" и "Requester" используются в технической документации
- В пользовательском интерфейсе используются более понятные формулировки
- Платформа не является агрегатором перевозок и не предоставляет услуги такси
- Все взаимодействия происходят между равноправными участниками совместных поездок
